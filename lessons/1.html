Lorsque vous naviguez sur le web, les bases de données sont partout et ce, sans que vous vous en rendiez compte.

Ici, lorsque vous cliquez sur le bouton pour ajouter au panier, les données relatives au produit sont stockée dans une table nommées "panier".

Une base donnée est composée de table

Dans quel format est enregistré une table ?

sql et xls étaient les bonnes réponses. En effect, une table est un peu comme un tableau excel.

Elle est composée de :

  • Colonnes
  • Lignes
  • Cellules

Voici un exemple pour notre table "panier" :

Quel colonne permet d'identifier le produit ?

C'est en effet la colonne pd_id qui permet l'identification du produit. Par aileur, cet élement permet de retrouver la produit dans une autre table (produits).

Il est temps maintenant d'introduire la notion de clef primaire et de clef etrangère

Quelle pourrait-être la clef primaire ici ?

Une clef primaire est unique, contrairement à une clef extrangère qui peut ne pas l'être.

Une clef extrangère provient d'une autre table où celle-ci est unique. Ainsi la clef user_id est unique dans la table "utilisateur".
